Introdução ao Controle de Versão

Git é um sistema de controle de versão distribuído que permite acompanhar mudanças no código-fonte durante o desenvolvimento de software.

Por que usar Git?

  • Histórico completo de alterações
  • Trabalho em equipe facilitado
  • Reverter mudanças quando necessário
  • Ramificação e merge de código

Checklist de Introdução

Fundamentos do Git

Antes de começar, você precisa instalar e configurar o Git em sua máquina.

Instalação e Configuração Inicial

# Configurar usuário
git config --global user.name "Seu Nome"
git config --global user.email "seu.email@exemplo.com"

# Verificar configurações
git config --list

Conceitos Básicos

  • Repositório: Local onde o Git armazena todos os arquivos e histórico
  • Commit: Snapshot das mudanças no repositório
  • Branch: Ramificação do código para desenvolver funcionalidades isoladas
  • Merge: Combinação de branches

Workflow Básico do Git

O fluxo de trabalho básico no Git envolve: trabalhar no diretório, preparar as mudanças e fazer commit.

Comandos Essenciais

# Inicializar um repositório
git init

# Verificar status dos arquivos
git status

# Adicionar arquivos para staging
git add nome_do_arquivo
git add .  # Adiciona todos os arquivos

# Fazer commit das mudanças
git commit -m "Mensagem descritiva do commit"

# Ver histórico de commits
git log

Ramificação e Merge

Branches permitem que você desenvolva funcionalidades, corrija bugs ou experimente ideias sem afetar o código principal.

Comandos de Branch e Merge

# Listar branches
git branch

# Criar nova branch
git branch nome_da_branch

# Mudar para uma branch
git checkout nome_da_branch
# ou
git switch nome_da_branch

# Criar e mudar para nova branch
git checkout -b nova_branch

# Fazer merge de uma branch
git merge nome_da_branch

# Resolver conflitos de merge (se houver)

GitHub e Repositórios Remotos

GitHub é uma plataforma de hospedagem de código para controle de versão usando Git.

Trabalhando com Repositórios Remotos

# Clonar um repositório existente
git clone https://github.com/usuario/repositorio.git

# Adicionar um repositório remoto
git remote add origin https://github.com/usuario/repositorio.git

# Enviar commits para o repositório remoto
git push -u origin main

# Atualizar repositório local com alterações remotas
git pull origin main

# Ver repositórios remotos configurados
git remote -v

Configurando SSH para GitHub

# Gerar chave SSH
ssh-keygen -t ed25519 -C "seu_email@exemplo.com"

# Adicionar chave SSH ao agente
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519

# Copiar chave pública e adicionar às configurações do GitHub
cat ~/.ssh/id_ed25519.pub

Tópicos Avançados

Depois de dominar o básico, você pode explorar recursos avançados do Git.

Rebase

# Rebase interativo para modificar commits
git rebase -i HEAD~3

Stash

# Guardar mudanças temporariamente
git stash

# Listar stashes
git stash list

# Aplicar stash mais recente
git stash apply

# Criar uma branch a partir de um stash
git stash branch nova_branch

Git LFS (Large File Storage)

# Instalar Git LFS
git lfs install

# Rastrear arquivos grandes
git lfs track "*.psd"
git lfs track "*.zip"

Boas Práticas com Git

Seguir boas práticas é essencial para um uso eficiente do Git, especialmente em equipe.

Padrões de Commit

  • Escreva mensagens de commit claras e descritivas
  • Use o modo imperativo ("Adiciona funcionalidade" em vez de "Adicionando")
  • Mantenha commits atomicos (uma mudança por commit)

Estratégias de Branching

  • Git Flow: Branches main, develop, feature, release, hotfix
  • GitHub Flow: Fluxo simplificado com branches de feature e deploy contínuo
  • Trunk Based Development: Menos branches, commits diretos na main

Projeto Prático

Agora é hora de colocar em prática tudo o que você aprendeu!

Criar um Projeto Completo

  1. Crie um novo repositório local para um projeto
  2. Adicione arquivos e faça commits iniciais
  3. Crie uma branch para desenvolver uma nova funcionalidade
  4. Faça merge da branch de volta à main
  5. Crie uma conta no GitHub e faça push do projeto
  6. Simule trabalho em equipe criando conflitos e resolvendo
