Com certeza! Aqui está um guia completo, passo a passo, para configurar o Git e o Git LFS (Large File Storage) para gerenciar arquivos grandes em seus projetos no GitHub.

O Git LFS é uma extensão do Git que substitui arquivos grandes por ponteiros de texto dentro do repositório, enquanto armazena o conteúdo real desses arquivos em um servidor remoto, como o próprio GitHub. Isso ajuda a manter o repositório Git leve e rápido.

Instalar Git (se necessário)

bash 
# Ubuntu/Debian
sudo apt-get install git

# Windows: Baixe e instale em https://git-scm.com/
# Mac: brew install git

Configurar usuário

bash 
git config --global user.name "Seu Nome"
git config --global user.email "seu.email@exemplo.com"

Configurar Conexão com GitHub

Gerar chave SSH (recomendado)

A chave SSH oferece uma forma segura de se conectar ao GitHub sem precisar digitar seu nome de usuário e senha toda vez.

bash 
ssh-keygen -t ed25519 -C "seu.email@exemplo.com"
# Pressione Enter para todas as perguntas, aceitando os padrões.

Adicionar chave ao GitHub

bash 
# Copiar chave pública para a área de transferência
cat ~/.ssh/id_ed25519.pub

# No GitHub: Vá para Settings → SSH and GPG keys → New SSH key
# Cole a chave copiada no campo "Key" e salve.

Testar conexão

bash 
ssh -T git@github.com

Você deverá receber uma mensagem de confirmação.

Para Projetos > 100MB - Usar Git LFS

Se o seu projeto inclui arquivos grandes como vídeos, áudios ou assets de design, o Git LFS é a solução ideal.

Instalar Git LFS

bash 
# Ubuntu/Debian
sudo apt-get install git-lfs

# Windows: Já vem incluído no Git for Windows
# Mac: brew install git-lfs

Configurar LFS no projeto

bash 
# Inicializar LFS no seu repositório local
git lfs install

# Especificar os tipos de arquivo que o LFS deve rastrear.
# Você pode adicionar quantos tipos precisar.
git lfs track "*.psd"
git lfs track "*.zip"
git lfs track "*.pdf"
git lfs track "*.mp4"
git lfs track "*.mov"
git lfs track "*.avi"
git lfs track "*.iso"
git lfs track "*.dmg"

# Para rastrear todos os arquivos em uma pasta específica
git lfs track "data/**"

Ver arquivos sendo rastreados

bash 
git lfs track

Criar e Configurar Repositório

No terminal

bash 
# Inicializar um novo repositório Git na sua pasta de projeto
git init

# Adicionar o arquivo .gitattributes, que o Git LFS cria para rastrear os arquivos grandes
git add .gitattributes

# Adicionar todos os outros arquivos do projeto
git add .

# Fazer o primeiro commit
git commit -m "Initial commit with Git LFS setup"

No GitHub

1. Crie um novo repositório no GitHub (não inicialize com README).

2. Copie a URL do repositório, que será algo como git@github.com:seu-usuario/nome-repositorio.git.

Conectar repositório local ao remoto

bash 
git remote add origin git@github.com:seu-usuario/nome-repositorio.git
git branch -M main
git push -u origin main

O git push -u origin main envia todos os seus arquivos (incluindo os arquivos grandes via LFS) para o repositório remoto.

Verificação e Troubleshooting

Verificar arquivos LFS

bash 
git lfs ls-files

Se já comitou arquivos grandes sem LFS

Se você acidentalmente comitou um arquivo grande antes de configurar o LFS, você precisará removê-lo do histórico do Git.

bash 
# Remover arquivos do cache do Git, mas mantendo-os localmente
git rm --cached arquivo-grande.zip

# Adicionar o tracking LFS
git lfs track "*.zip"

# Fazer um novo commit e push
git add arquivo-grande.zip
git commit -m "Move large file to LFS"
git push

Limite do GitHub LFS

O GitHub oferece um plano gratuito com:

  • 1 GB de armazenamento gratuito.
  • 1 GB de banda por mês gratuita.

Para projetos que excedem esse limite, você pode precisar adquirir pacotes de dados ou considerar alternativas como o GitLab, que possui um limite de LFS mais generoso, ou hospedar seu próprio servidor Git.

Com essa configuração, você estará pronto para gerenciar projetos com arquivos grandes de forma eficiente, mantendo seu repositório Git organizado e leve!
